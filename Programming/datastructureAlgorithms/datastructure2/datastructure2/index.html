# 数组

数组在内存中是按顺序存储的，数组内每个元素都有一个单独的下标，在查找元素时只需要给出对应的下标即可，下标就是一个数字，从第一个元素为 0 逐次增加，这叫随机读取，读取的时间复杂度只有 O (1)
数组的长度在创建数组的时候就固定了，所以当长度不够时就只能再创建一个更长的数组再将元素一个个复制过去（Java 是这样的，在 JavaScript 中是动态的可随意增减）扩容的时间复杂度是 O (n)
由于每个元素是固定下标，所以在数组中作中间插入的操作是繁琐的，需要把要插入的位置的元素包括在这之后的元素一个个往后移，中间删除操作往前移，时间复杂度都是 O (n)
如果没有顺序要求，中间删除操作可以将最后一个元素复制到要删除元素的位置，然后删除最后一个元素，这样时间复杂度只有 O (2)

# 总结

数组的随机读取能力强但插入、删除能力会导致大量元素移动，因此数组适合在读操作多，写操作少的情况使用

# 链表

# 单向链表

每一个节点由 data 、 next 组成， data 为存储的数据， next 指向下一个节点，头节点为 head ，尾部节点为 last 且 next 指向 null ，因此只能做向下读取的操作
链表是随机存储的 (所以需要指针)，在做查找的操作时只能一个一个节点往下找，时间复杂度为 [O (n)].{pink}，但它的长度是动态的
头部插入：将新节点的 next 指向第一个节点，然后更新 head 为新节点
中间插入：将新节点的 next 指向下一个节点，将上一个节点的 next 指向新节点
头部删除：将 head 更新为第二个节点（Java 特性会删除未被使用的数据）
中间删除：将上一个节点的 next 指向下一个节点
尾部删除：将倒数第二个节点的 next 指向 null

# 双向链表

和单向链表唯有一点不同，一个节点多包含了 prev ，用于指向上一个节点

# 总结

如果忽略查找的过程，链表在插入、删除方面上更为灵活，链表适合用于在尾部频繁插入、删除的情况下
